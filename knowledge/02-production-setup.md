# Production Setup Guide

## Prerequisites

- Docker and Docker Compose v2+
- Python 3.11+ (for local development)
- An OpenAI API key
- A Vapi account + API key + phone number (for live voice calls)
- A public URL for Vapi webhooks (ngrok for development)

---

## Step 1: Environment Configuration

Copy the example env and fill in your values:

```bash
cp .env.example .env
```

Edit `.env`:

```env
# Required
OPENAI_API_KEY=sk-your-openai-key

# Voice (required for live calls, optional for simulated mode)
VAPI_API_KEY=your-vapi-api-key
VAPI_PHONE_NUMBER_ID=your-vapi-phone-number-id
VAPI_WEBHOOK_URL=https://your-public-url/webhook/vapi

# Temporal (defaults work for Docker Compose)
TEMPORAL_HOST=localhost:7233
TEMPORAL_NAMESPACE=default
TEMPORAL_TASK_QUEUE=collections

# API
API_HOST=0.0.0.0
API_PORT=8000

# Voice mode: "simulated" for testing, "live" for actual phone calls
VOICE_MODE=simulated
```

---

## Step 2: Running with Docker Compose (Recommended)

```bash
# Start all services
docker compose up -d

# Check logs
docker compose logs -f

# Verify health
curl http://localhost:8000/health
```

This starts:
- **PostgreSQL** (Temporal persistence)
- **Temporal Server** (port 7233)
- **Temporal UI** (port 8080 — visit http://localhost:8080)
- **FastAPI** (port 8000)
- **Temporal Worker** (processes workflows)

All services should be healthy within ~2-3 minutes.

---

## Step 3: Running Locally (Development)

```bash
# Install dependencies
pip install -e ".[dev]"

# Start Temporal (Docker only for infra)
docker compose up -d postgresql temporal temporal-ui

# In terminal 1: Start API
make dev-api

# In terminal 2: Start Worker
make dev-worker
```

## Step 4: Running the Learning Loop

The learning loop runs offline and does not require Temporal or Docker.

```bash
# Ensure .env has OPENAI_API_KEY set
make learn
```

This will:
1. Initialize prompt versions in `prompts/` directory
2. Run 5-8 iterations of simulate → evaluate → mutate → compare
3. Save all data to `data/evaluations/` and `data/reports/`
4. Generate cost report in `data/reports/cost_report.json`
5. Stop when budget ($20) is exhausted or max iterations reached

**Expected runtime:** 15-30 minutes depending on API speed.
**Expected cost:** $6-8.

To generate the evolution report:

```bash
make report
```

## Step 5: Testing a Live Conversation

```bash
# 1. Start a workflow for a borrower
curl -X POST http://localhost:8000/workflow/start \
  -H "Content-Type: application/json" \
  -d '{
    "borrower_id": "B001",
    "name": "John Doe",
    "account_last4": "7823",
    "total_debt": 4500.00,
    "debt_type": "credit_card",
    "days_past_due": 90,
    "phone_number": "+15551234567",
    "email": "john@example.com"

# 2. Check workflow status
curl http://localhost:8000/workflow/B001/status

# 3. Send a chat message (as the borrower)
curl -X POST http://localhost:8000/chat/B001 \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello, I got a message about my account?"}'

# 4. Continue the conversation
curl -X POST http://localhost:8000/chat/B001 \
  -H "Content-Type: application/json" \
  -d '{"message": "Yes, my last 4 digits are 7823"}'
```

## Step 6: Vapi Voice Setup (Live Calls)

For live voice calls with the Resolution agent:

1. Sign up at [vapi.ai](https://vapi.ai)
2. Create a phone number in your Vapi dashboard
3. Set `VOICE_MODE=live` in `.env`
4. Set `VAPI_API_KEY`, `VAPI_PHONE_NUMBER_ID`
5. Expose your webhook URL publicly (e.g., with ngrok):

```bash
ngrok http 8000
# Copy the https URL to VAPI_WEBHOOK_URL in .env
```

6. Restart the services:

```bash
docker compose restart api worker
```

When the workflow transitions from Assessment to Resolution, the system will initiate an outbound call via Vapi to the borrower's phone number.

## Step 8: Prompt Management

View and manage prompt versions via the admin API:

```bash
# List all prompt versions for assessment agent
curl http://localhost:8000/admin/prompts/assessment

# Get the currently active prompt
curl http://localhost:8000/admin/prompts/assessment/active

# Rollback to a previous version
curl -X POST http://localhost:8000/admin/prompts/assessment/rollback/{version_id}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Temporal worker can't connect | Ensure Temporal server is healthy: `docker compose logs temporal` |
| OpenAI rate limits | Reduce `conversations_per_eval` in config or add delays |
| Vapi calls not working | Check `VAPI_WEBHOOK_URL` is publicly accessible, check Vapi dashboard logs |
| Token budget errors | Run `make test` to verify prompts fit within limits |
| Learning loop stops early | Check `data/reports/cost_report.json` for budget status |
